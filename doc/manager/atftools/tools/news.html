ATF Tools Release Notes

version 1.9e, 10/28/05

  • atfcheck.plx now traps multiply defined P-IDs
  • atfsquare.plx more bug fixes; output is now reliable but should still be diff'd
  • atf2xtf.plx new option -digest
  • ATF LibXML ID fixes: ID generation is now 100% stable and unique
  • ATF Parser bug with ~x fixed

version 1.9d, 10/28/05

  • Fix bug which resulted in bad line ids in .xlk files
  • Fix bug related to primes in line numbers; ASCII apostrophe is now mapped to U+2032 by atf2xtf.
  • atfcheck.plx now validates @column values
  • atfcheck.plx now traps misplaced square brackets li[ke th]is
  • atfsquare.plx bug fixes
  • Parser now handles backtick (`) as aleph in grapheme bodies
  • Unicode routines now translate x4 idiom correctly
  • CDL::ATF::GDL library bugfixes

version 1.9c, 10/21/05

  • Collection of improvements to support programs; no changes in atf2xtf
  • atfsquare.plx reimplementation (again!). Apparently actually works now
  • atfsquare.plx now understands [# ... #] for half-bracket conversion
  • atf2ascii.plx now converts accented vowels to numeric indices
  • atf2ascii.plx now converts h+rocker to h and left/right ceiling to [#/#]
  • atfgrabpq.plx now has several options
  • atfsplit.plx, atfcheck.plx several minor bug fixes
  • atftriage.plx improved support for triaging lemmatized ATF
  • atflog.plx is a new program to help convert logograms

version 1.9b, 9/19/05

  • Remove dependency on String::CRC32 for IDs; new ID system fully serviceable (should never produce bad XML) but needs minor tweaks
  • Fix bug in @div handling that affected atf2xtf and atfcheck
  • Eagles beat SF 42-3

version 1.9a, 9/16/05

  • Fixes to tests; distribution should now pass all tests
  • Minor bugfixes for processing multiple files

version 1.9, 7/16/05

  • Various bug-fixes occasioned by getting the DCCLT P/Q files down to 0 errors
  • atfcheck.plx now detects several more errors including invalid text links
  • #atf use lexical now automatically implies #atf use mylines

version 1.8, 6/16/05

  • xml:id's produced by atf2xtf.plx are now unique system-wide
  • New programs atf2ascii.plx, atfgrabpq.plx
  • Serializer no longer produces empty words for <<...>>
  • atf2xtf.plx now produces W nodes for standalone ellipsis
  • atf2xtf.plx now handles multiple glosses ({disz}{d}) and hyphenated glosses ({u-du}udu) correctly
  • Various fixes to atflemma.plx

version 1.7, 6/15/05

  • Interface change to atf2xtf.plx: reading STDIN now means reading a list of files to process (used to mean read input to process but that was never used to my knowledge).
  • Number bugfix in ATF/Parser; {gi} determinative no longer considered a unit
  • Numerous additions, improvements and fixes to atflemma.plx
  • Add -list option to atfsplit.plx
  • Add -lemma option to atftriage.plx
  • Code cleanups and -alpha option in atfwords.plx

version 1.6, 6/02/05

  • Add new program atflemma.plx
  • Add link to atf2gdl.plx documentation
  • Add -verbose option to atfwords.plx
  • Lang/RWS bugfixes to ATF Parser (and Tokenizer/Serializer)
  • Various small bugfixes to atftokens.plx, atfchars.plx and atftriage.plx
  • Update documentation in programs and web docs

version 1.5, 5/25/05

  • Implement understanding of #lem: comments in ATF files to prepare for proper XTF lemmatization
  • Add new programs atfchars.plx, atfcase.plx
  • Reimplement atfsquare.plx with new algorithm which works properly for more cases

version 1.4, 5/23/05

  • Additional features--see xtf.dtd until ATF docs are updated
  • Fix bug in _..._ processing

version 1.2, 6/31/04

Minor bug fixes related to archaic text processing.

version 1.1, 6/29/04

Add atftriage.plx and atf2xtf support for emptying invalid files.

version 1.0, 6/24/04

Initial release.
