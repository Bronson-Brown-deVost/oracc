Oracc Documentation

Oracc recognizes several roles people can fulfill when interacting with the system. A single person may act in more than one role. At the top level, the documentation is divided up according to the roles to which it is principally relevant.

User

User is short for end-user, someone who is working with Oracc for study.

Builder

A builder is someone who is working on texts actually helping to build Oracc—typing up data or lemmatizing, for example.

Manager

A manager is someone who helps administer an Oracc project and who therefore needs to understand aspects of the system that builders and users don't need to care about.

Developer

A developer is someone who is able to program Oracc at some level—writing XQueries for the database, or style files for the OpenDocument support, for example.

Developers may also be interested in the the Schema documentation.

Sysadmin

Oracc typically has several system administrators—people who are able to create new projects, edit the documentation, restart the various services.

Sysadmin documentation is only accessible to sysadmins.

Steerer

The Oracc Steering Committee consists of a core of senior Oracc users who are responsible for oversight and direction of Oracc. Steerers are always sysadmins, but sysadmins do not necessarily always need to be steerers.

Steerer documentation is only accessible to steerers.
