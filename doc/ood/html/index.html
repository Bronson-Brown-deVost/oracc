OOD: Oracc Open Data

Steve Tinney
Version of 2012-11-16

Introduction

This document provides an introduction to Open Data on Oracc and also serves as a guide to the various pieces of documentation about the OOD.

The implementation of comprehensive, systematic Open Data access for Oracc is planned as a multi-phase operation. In the first phase the intention is to provide stable, consistently organized web addresses for all of Oracc's most important data. Future phases will increase the extent of this implementation and add semantically-based access and discovery.

Concepts

In order to understand Open Data it is useful to be familiar with the following basic concepts and how Oracc understands them:

Resource
Any kind of data point in Oracc from a project to a word instance. The resources involved in OOD are described in the OOD resources document.
URN: Uniform Resource Name
a subset of URIs which define a persistent, unique name for a resource without implying anything about any possible locations of the resource. The Oracc URN scheme is described in the OOD URN document.
URI: Uniform Resource Identifier
a URI is a string which identifies a name or resource. In the OOD documentation we generally use the term URI in a restricted sense to refer to the http-based URIs which provide the public access methods to OOD and which are described in the OOD URI document.
URL: Uniform Resource Locator
a subset of URIs which is used in Oracc only to refer to file-resources which are referred to directly by the URL.

Reading Order

If you are new to the OOD you will likely benefit most from reading the documentation in the following order:

  • Oracc URI Patterns
  • Oracc Resources
  • OOD URNs
  • OOD URIs
  • URIs for texts

The Patterns document does depend on an understanding of the Resources, URNs and URIs documents, but nevertheless some familiarity with the patterns will probably clarify the verbiage of the other pages.
