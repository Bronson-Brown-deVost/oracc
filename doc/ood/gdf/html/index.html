GDF: Generic Data Framework

Steve Tinney
Version of 2013-01-06

Introduction

GDF provides a way of turning arbitrary datasets into linked open data quickly and easily.

GDF is designed with an emphasis on simplicity: the data is minimally processed and accessible via a general full-text search engine. It is particularly well-suited to rescuing legacy data from obscurity, and as such users should expect that the data will often be unpolished.

We encourage users of GDF datasets to think about similar works they may have created as part of their research and which they might think of as useful but imperfect. Please send us such datasets with written permission (e-mail is fine) to convert and distribute them! We will reformat them as necessary and distribute them under the Creative Commons BY-SA licence.

Using a GDF Dataset

GDF datasets are easy to use: you should read the home-page for any information on abbreviations, conventions, etc., and then you can normally browse some fields to get a sense of what to search for.

Technical Notes

At the very least, the data will have been structured into records with stable XML IDs which are then used in the URIs to provide persistent access. The data will also be in the Unicode character set in UTF-8 encoding.

GDF datasets generally provide their records in XML format if you add /xml to the URI. If the status-field stable-uris is set to yes (as it normally will be) you can safely refer to the data by the URI and know that the URI will not disappear.

Future Plans

While GDF prides itself on its simplicity, there are plans for a few key improvements:

  • improved formatting of records--the present simple list of fields and data will be supplemented by a more database-like layout option
  • a facility to get a row-oriented view of the data and search results, and to browse the pages of data or search results in order.

There is no timetable for these developments.
